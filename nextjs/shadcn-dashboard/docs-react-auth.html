React JS — Authentication

Authentication

The React JS version supports multiple authentication methods. By default, authentication is handled client-side using form-based login. You can switch to JWT, Firebase, or Supabase by updating the platform value in AuthContext.tsx.

By Default, we are using Firebase for auth, if you want to change it to Jwt or Supabase for that follow below steps.
Select the jwt-firebase Demo folder
// /reactjS/firebase
npm install

Global App Setup with Authentication & Theme

The app entry point wraps the entire app with ThemeProvider and CustomizerContextProvider. MSW mock workers are started before React renders so all API calls are intercepted from the start. 

// src/main.tsx
import { Suspense } from 'react';
import { createRoot } from 'react-dom/client';
import './css/globals.css';
import App from './App.tsx';
import Spinner from './views/spinner/Spinner.tsx';
import { CustomizerContextProvider } from './context/CustomizerContext.tsx';
import './utils/i18n';
import { ThemeProvider } from './context/shadcntheme/ThemeContext.tsx';
import { AuthProvider } from 'src/guards/auth/AuthContext.tsx';      

createRoot(document.getElementById('root')!).render(
    <ThemeProvider defaultTheme="system" storageKey="activeMode">
      <CustomizerContextProvider>
        <Suspense fallback={<Spinner />}>
         <AuthProvider>
           <App />
         <AuthProvider/>
        </Suspense>
      </CustomizerContextProvider>
    </ThemeProvider>
);

1. JWT Method

To use JWT authentication, update the platform value in your auth context to 'JWT'. Connect handleSubmit in AuthLogin.tsx to your backend JWT endpoint. 

// src/guards/auth/AuthContext.tsx
const initialState = {
  isAuthenticated: false,
  isInitialized: false,
  user: null,
  platform: 'JWT',   // <-- change to 'JWT'
};
Platform Configuration
// -----------------------------------------------------------------------------------------
// File : src/guards/auth/AuthContext.tsx
// -----------------------------------------------------------------------------------------
const initialState: InitialStateType = {
  platform: 'JWT', // Change platform to JWT for JWT Authentication
};

2. Firebase Method

To use Firebase authentication, add your Firebase config to src/guards/firebase/Firebase.tsx and set platform: 'Firebase' in the auth context. 

// src/guards/firebase/Firebase.tsx
const firebaseConfig = {
  apiKey:            'API_KEY',
  authDomain:        'AUTH_DOMAIN',
  projectId:         'PROJECT_ID',
  storageBucket:     'STORAGE_BUCKET',
  messagingSenderId: 'SENDER_ID',
  appId:             'API_ID',
  measurementId:     'M_ID',
  databaseURL:       'DATABASE_URL',
};
Steps to Generate Firebase Key
  1. Go to firebase.google.com and open the Console
  2. Click Add Project, enter a project name and click Create Project
  3. In the left panel go to Build > Authentication
  4. Click Web Setup and copy the config object
  5. Paste it into src/guards/firebase/Firebase.tsx
  6. Enable Email/Password sign-in method in the Authentication tab

Reference: https://firebase.google.com/

Steps for Generate key
  • Click on "Go to console" displayed on the right side on your firebase account.
  • Then after click on Add Project.
  • After the popup menu appears, enter a Project name, select a Country, accept Terms and Conditions, then click Create Project.
  • Now the Firebase Admin panel will display.
  • In the left panel, click on Develop. Now click on Authentication link.
  • Click on the "Web Setup" button.
  • Copy the configuration and replace it in your project at src/components/Firebase/firebase.js.
  • Go to the Sign-in Method tab. Enable Email/Password and Anonymous in sign-in providers.
  • Now click on Database link from the left panel.
  • Click on the Create a database button.
  • In the popup, select "start in locked mode" and click Enable.
  • Now select Real-time Database.
  • Click on the Rules tab. Change the read and write rules from false to true.

3. Supabase Method

To use Supabase auth, install the Supabase client, add your credentials to .env, create a supabaseClient.ts file, and set platform: 'Supabase'.

Environment File .env
VITE_SUPABASE_URL="YOUR_VITE_SUPABASE_URL"
VITE_SUPABASE_ANON_KEY="YOUR_VITE_SUPABASE_ANON_KEY"
Supabase Client — src/guards/supabase/supabaseClient.ts
import { createClient } from '@supabase/supabase-js';

const supabaseUrl: string = import.meta.env.VITE_SUPABASE_URL;
const supabaseKey: string = import.meta.env.VITE_SUPABASE_ANON_KEY;

export const supabase = createClient(supabaseUrl, supabaseKey);
Steps to Generate Supabase Keys
  1. Visit supabase.com and sign up or log in
  2. Click New Project on the dashboard
  3. Enter a project name, select a region and click Create Project
  4. Go to Authentication tab — Email/Password is enabled by default
  5. Optionally enable OAuth providers: Google, GitHub, Twitter etc.
  6. Click API in the left sidebar
  7. Copy the Project URL → paste as VITE_SUPABASE_URL
  8. Copy the Anon Key → paste as VITE_SUPABASE_ANON_KEY
  9. Install the client: npm install @supabase/supabase-js
  10. Create the .env file and supabaseClient.ts as shown above
Platform Configuration
// -----------------------------------------------------------------------------------------
// File : src/guards/auth/AuthContext.tsx
// -----------------------------------------------------------------------------------------
const initialState: InitialStateType = {
  platform: 'Supabase', // Change platform to Supabase for Supabase Authentication
};
Steps for Generate Supabase Key
  • Go to https://supabase.com/
  • Click on "Sign Up" or "Log In" and complete the process.
  • On the dashboard, click on the "New Project" button.
  • Enter Project name, select a Region, then click Create Project.
  • Once your project is created, go to the Project Dashboard.
  • In the left panel, click on the Authentication tab.
  • By default, Email & Password authentication is enabled.
  • You can also enable OAuth providers like Google, GitHub, Twitter, etc.
  • In the Settings tab under Authentication, you can customize email templates for verification and password reset.
  • From the left sidebar, click on API.
  • Project URL: Copy this URL and use it as VITE_SUPABASE_URL.
  • Anon Key: Copy this key and use it as VITE_SUPABASE_ANON_KEY.
  • Install Supabase: npm install @supabase/supabase-js
  • Create an .env file to store your keys: 
    // this is your .env file
VITE_SUPABASE_URL="Your Url"
VITE_SUPABASE_ANON_KEY="Your SUPABASE_ANON_KEY Key"
  • Create the supabaseClient.ts file: 
    import { createClient } from '@supabase/supabase-js';

const supabaseUrl: string | any = import.meta.env.VITE_SUPABASE_URL;
const supabaseKey: string | any = import.meta.env.VITE_SUPABASE_ANON_KEY;

export const supabase = createClient(supabaseUrl, supabaseKey);
