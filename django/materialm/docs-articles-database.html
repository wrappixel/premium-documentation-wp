    Django Admin - Database Setup

    MaterialM Django Template fully supports multiple databases such as SQLite, PostgreSQL, and Neon Db. By default, the project uses Djangos built-in SQLite for quick setup, but you can easily switch to any production-ready database.

    1. Integrate Manually with SQLite

    SQLite is the default database for Django and works out-of-the-box with MaterialM. Use this section when you want a quick local setup or when working on small projects.

    Step 1 Verify settings.py

    By default Django points to an SQLite file in your project root. Confirm the following in settings.py: 

    
DATABASES = {
    "default": {
        "ENGINE": "django.db.backends.sqlite3",
        "NAME": BASE_DIR / "db.sqlite3",
    }
}

    You can configure your preferred database inside settings.py DATABASES.

    Step 2 Create models

    Add your model(s) inside the app's models.py. Example: 

    
# transactions/models.py
class Transaction(models.Model):
    title = models.CharField(max_length=200)
    amount = models.DecimalField(max_digits=10, decimal_places=2)
    date = models.DateField()
    ...

    Step 3 Run migrations

    Generate and apply migrations to create the SQLite tables: 

    
python manage.py makemigrations
python manage.py migrate

    Step 4 Optional: view data in admin

    Register your model in admin.py and create a superuser to manage data: 

    
python manage.py createsuperuser

    Once complete, your MaterialM project will be using SQLite and ready for CRUD pages.

    2. Other Integrations (Postgres / Neon Db) & Load JSON

    For production and larger projects you may switch to PostgreSQL or MySQL. This section shows required drivers, sample DATABASES settings, and how to load MaterialM demo JSON fixtures.

    PostgreSQL

    Install the driver and update settings.py: 

    
pip install psycopg2-binary


    
DATABASES = {
    "default": {
        "ENGINE": "django.db.backends.postgresql",
        "NAME": "your_db_name",
        "USER": "your_db_user",
        "PASSWORD": "your_db_password",
        "HOST": "db_host_or_ip",
        "PORT": "5432",
    }
}

    Apply migrations

    After configuring the database run: 

    
python manage.py makemigrations
python manage.py migrate

    Load MaterialM Demo JSON (fixtures)

    MaterialM ships demo data as JSON fixtures. To load them: place the JSON file inside the app's fixtures/ folder, then run: 

    
python manage.py loaddata path/to/your/fixture.json

    Example location: transactions/fixtures/transactions.json

    Tips & Best Practices

    • Use environment variables (.env) for DB credentials (do not commit them).
    • Keep separate settings for development and production (DATABASES per env).
    • Back up data using dumpdata before switching DBs: python manage.py dumpdata > backup.json
    • Run migrations in a staging environment first for production DBs.

    With these steps, you can integrate MaterialM with any SQL database and populate it using the included JSON fixtures for quick previews and testing.
