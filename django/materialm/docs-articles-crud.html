    CRUD Documentation

    CRUD stands for Create, Read, Update, and Delete. These essential operations allow you to manage data effectively in any application. Our MaterialM Django Template provides a clean and optimized CRUD structure using Django's Class-Based Views, combined with LoginRequiredMixin and LayoutMixin to ensure all pages follow a consistent layout.

    CRUD pages come with pre-built UI, Django form handling, validation, layout helpers, and support for flash messages.

    Introduction to CRUD Operations

    CRUD operations are the foundation of any data-driven application. They enable you to create new entries, retrieve existing records, modify them, or remove them completely. This section explains how CRUD works inside the MaterialM Django Template.

    What is a Database?

    A database stores your application data—such as users, products or transactions. MaterialM uses Django’s built-in SQLite database by default, which is lightweight and perfect for small to medium projects. You can switch to other databases anytime by updating Django settings.

    Tip: MaterialM includes example CRUD logic inside the transactions module. You can follow the same structure to build your own CRUD modules.

    Setting Up CRUD Database

    Before working with CRUD operations in MaterialM, you must create a Django app, define models, and apply migrations to generate database tables. MaterialM uses Django’s built-in SQLite database by default, which makes setup fast and simple.

    Tip: This is a basic example showing how database tables are created in MaterialM.

    1. Create a Django Project

    Use the following command to create a new Django project:

    Note: You don't need to do this step as MaterialM is already a Django Project so go with 2

    
django-admin startproject myproject

    2. Create a Model

    Inside your app (e.g., transactions), define your model in models.py. A model represents the structure of your database table. 

    
# models.py
from django.db import models

class Order(models.Model):
    order_id = models.CharField(max_length=20)
    customer_name = models.CharField(max_length=100)
    ...

    3. Register Model in Admin (Optional)

    To view and manage your records via Django Admin, register your model: 

    
# admin.py
from django.contrib import admin
from .models import Order

class OrderAdmin(admin.ModelAdmin):
      ...

admin.site.register(Order)

    4. Run Migrations

    Django uses migrations to create and modify database tables. Run these commands after adding or updating models: 

    
python manage.py makemigrations
python manage.py migrate

    Your database tables are now ready. You can begin creating CRUD views using MaterialM’s structure and helpers.

    1. Create

    Create operation is used to add new records into the database.
    In MaterialM, we use Django’s CreateView combined with LoginRequiredMixin and LayoutMixin to auto-manage layout, breadcrumbs, and page titles.

    Implementation: The create logic is handled inside transactions/add/views.py where we validate form input and save a new record in the database. 

    
class TransactionCreateView(LoginRequiredMixin, LayoutMixin, CreateView):
    model = Order
    form_class = OrderForm
    template_name = "transactions_add.html"
    success_url = reverse_lazy("transactions-list")
    ...

    2. Read (List & Detail)

    Read operation displays data using ListView and DetailView.
    MaterialM provides modern UI tables, pagination support, filters, and clean layout for listing records.

    Implementation: Data listing and filtering logic is implemented inside transactions/views.py. We fetch user-based data and pass it to templates for rendering admin/client views.

    3. Update

    Update allows users to modify existing data.
    UpdateView loads the selected object and automatically builds the edit form.

    Implementation: Update logic is implemented in transactions/edit/views.py. The view loads the object, binds form data, and saves changes with validation. 

    
class TransactionUpdateView(LoginRequiredMixin, LayoutMixin, UpdateView):
    model = Order
    form_class = OrderForm
    template_name = "transactions_edit.html"
    success_url = reverse_lazy("transactions-list")
    pk_url_kwarg = "id"
    ...

    4. Delete

    Delete operation removes a specific record from the system.
    MaterialM includes a confirmation popup/modal to prevent accidental deletion.

    Implementation: Delete logic exists in transactions/views.py and uses a confirmation modal before final deletion.

    All CRUD templates follow the MaterialM UI structure and are placed inside:
    main/
    You can fully customize forms, tables, and layout based on your project needs.
